<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| admin browser -> Server Actions | Untrusted form data writes operational setup rows. |
| content catalog -> database assignment | Repo-owned service IDs become database references. |
| admin UI -> future public availability | Invalid setup rows would poison later public slot calculation. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05 | Elevation of privilege | `lib/booking/setup-actions.ts` | mitigate | Every exported action starts with `await requireAdmin()` from Plan 02-01. |
| T-02-06 | Tampering | `lib/booking/setup-validation.ts` | mitigate | Zod validates staff input and service assignments; service IDs are checked against `bookableServices`. |
| T-02-07 | Information disclosure | `lib/booking/setup-queries.ts` | mitigate | Return setup DTOs only; never query or expose admin credential fields. |
| T-02-08 | Tampering | `staff_services` writes | mitigate | Transactionally replace assignments and persist only service IDs, preventing stale duplicated title/price/duration data. |
</threat_model>
