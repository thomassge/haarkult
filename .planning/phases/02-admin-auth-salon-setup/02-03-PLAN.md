<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| admin browser -> schedule Server Actions | Untrusted weekly range and exception form data writes availability records. |
| setup database -> Phase 3 availability engine | Persisted setup rows will determine public slots. |
| local date input -> timestamptz persistence | Admin local-day intent must become unambiguous stored windows. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-09 | Elevation of privilege | `lib/booking/setup-actions.ts` | mitigate | Every weekly and exception action calls `requireAdmin()` before parsing or writing. |
| T-02-10 | Tampering | `lib/booking/setup-validation.ts`, `db/schema.ts` | mitigate | Validate weekday/minute/window rules with Zod and add DB check constraints for range validity. |
| T-02-11 | Denial of service | weekly availability overlap handling | mitigate | Reject overlapping ranges before writes so Phase 3 does not generate duplicate or explosive slot sets. |
| T-02-12 | Tampering | exception normalization | mitigate | Normalize all-day Europe/Berlin windows and require timed `endAt > startAt` before persistence. |
</threat_model>
