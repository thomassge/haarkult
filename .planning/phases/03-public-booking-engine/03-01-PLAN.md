<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Public browser to booking route state | Client-side service and stylist choices are untrusted and must be re-derived from catalog and setup data. |
| Public route to database-backed setup reads | Public users may see only safe staff names and service options, not admin credentials or setup diagnostics. |
| Contact-only mode to booking controls | Product mode must prevent booking controls from appearing when the salon is contact-only. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01-01 | Tampering | `deriveStylistPreferenceOptions` | mitigate | Derive eligibility from active staff plus assigned service ids for the selected catalog service; never trust a browser-supplied staff id as eligible. |
| T-03-01-02 | Information Disclosure | setup-incomplete fallback | mitigate | Show only German contact fallback copy and actions; do not render setup diagnostics, database errors, or admin route names. |
| T-03-01-03 | Elevation of Privilege | public route imports | mitigate | Keep public booking route free of `requireAdmin()`, admin mutation imports, and admin credential modules. |
</threat_model>
