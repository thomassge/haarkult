# Phase 03 - UI Design Contract

> Visual and interaction contract for the public booking engine. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none - manual project UI primitives |
| Preset | not applicable |
| Component library | none; use existing `components/ui/*` primitives with route-local overrides where needed |
| Icon library | none installed; do not add icons unless implementation already has a dependency |
| Font | Geist Sans via `app/layout.tsx` and `app/globals.css` |

Design system detection: `components.json` is absent, `postcss.config.mjs` and Tailwind CSS 4 are present, and the repo already uses custom primitives in `components/ui/`. shadcn gate result: do not initialize shadcn for Phase 03; registry safety is not applicable.

Phase-specific override: the booking route must not reuse large rounded marketing cards unchanged. Interactive booking panels, chips, form fields, and buttons use `8px` radius or less, no card-inside-card layouts, and no decorative gradient orb or bokeh backgrounds.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline gaps between text and metadata |
| sm | 8px | Chip gaps, field helper spacing, radius ceiling |
| md | 16px | Default element spacing, form field stacks |
| lg | 24px | Step groups, panel padding on mobile |
| xl | 32px | Desktop panel padding, main grid gaps |
| 2xl | 48px | Major state changes inside the flow |
| 3xl | 64px | Top and bottom page spacing on desktop |

Exceptions: touch targets for chips, buttons, date cells, time slots, and form controls must be at least `44px` tall. Desktop main content may use `40px` horizontal container padding only where inherited from `Container`; new booking internals stay on the declared scale.

---

## Typography

Use exactly these four sizes in the booking flow. Override inherited negative tracking with `tracking-normal` or equivalent `letter-spacing: 0`.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 |
| Label | 14px | 600 | 1.4 |
| Heading | 20px | 600 | 1.2 |
| Display | 28px | 600 | 1.2 |

Weights allowed: `400` and `600` only. Do not use uppercase tracking-heavy eyebrow text inside the booking app; it increases visual noise in a transactional flow.

## Color

Phase 03 uses a route-local neutral and green contract so the booking experience does not read as a beige/brown or dark slate interface.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#f7f8f6` | Page background and unframed route surface |
| Secondary (30%) | `#ffffff` | Booking panels, form fields, date rows, summary surface |
| Accent (10%) | `#23624f` | Primary CTA, selected category/service/stylist/date/slot, focus ring, current step marker, success status |
| Destructive | `#b42318` | Validation errors and destructive actions only |

Accent reserved for: primary submit button, selected option border/background, current progress step, focus ring, success confirmation badge, and retry action after a stale-slot conflict. Do not use accent for every link, every icon, or decorative backgrounds.

Supporting colors:

| Role | Value | Usage |
|------|-------|-------|
| Text primary | `#161a17` | Main copy and selected labels |
| Text muted | `#5f6b62` | Metadata, helper copy, disabled text |
| Border | `#d9e1da` | Panel, field, chip, and slot boundaries |
| Warning/empty tint | `#eef4ef` | No-slot and setup fallback information blocks |

## Copywriting Contract

All user-facing copy is German only. Keep copy direct and transactional; no account prompts, no technical setup wording, and no legal block in the form.

| Element | Copy |
|---------|------|
| Primary CTA | `Termin anfragen` in manual confirmation mode; `Termin buchen` in instant confirmation mode |
| Empty state heading | `Keine freien Zeiten gefunden` |
| Empty state body | `Waehle ein anderes Datum, eine andere Leistung oder eine andere Stylistin aus.` |
| Error state | `Diese Zeit ist gerade nicht mehr verfuegbar. Deine Angaben bleiben erhalten. Bitte waehle eine neue Uhrzeit.` |
| Destructive confirmation | Keine destruktiven Aktionen in Phase 03 |

Required field labels:

| Field | Label | Helper or error copy |
|-------|-------|----------------------|
| Name | `Name` | Error: `Bitte gib deinen Namen ein.` |
| Phone | `Telefon` | Error: `Bitte gib eine Telefonnummer fuer Rueckfragen ein.` |
| Email | `E-Mail` | Error: `Bitte gib eine gueltige E-Mail-Adresse ein.` |
| Note | `Hinweis (optional)` | Helper: `Wuensche, Besonderheiten oder eine bevorzugte Rueckrufzeit kannst du hier eintragen.` |

Trust copy near contact fields: `Wir nutzen deine Angaben nur fuer diese Terminanfrage. Du brauchst kein Kundenkonto.`

Success copy:

| Mode | Heading | Body |
|------|---------|------|
| Manual | `Deine Anfrage ist angekommen` | `Der Salon prueft den Termin und meldet sich persoenlich bei dir.` |
| Instant | `Dein Termin ist gebucht` | `Der Termin ist eingetragen. Bitte komm ein paar Minuten vorher in den Salon.` |

Setup-incomplete fallback copy:

| Element | Copy |
|---------|------|
| Heading | `Termin direkt anfragen` |
| Body | `Online sind gerade keine Zeiten verfuegbar. Du erreichst den Salon direkt per Telefon, WhatsApp oder E-Mail.` |

## Layout Contract

`/termin-buchen` is the primary usable booking experience, not a marketing landing page. The first viewport must start with the booking flow itself.

| Area | Contract |
|------|----------|
| Page shell | Full-width neutral background, no hero image, no decorative orbs, no split marketing hero |
| Main layout | Mobile: single column. Desktop: two-column layout with the active step on the left and a sticky summary on the right |
| Max width | `76rem` using existing `Container`; booking internals must stay readable at 320px width |
| Active panel | One un-nested panel per step, `8px` radius, white surface, `1px` border |
| Summary | Desktop sticky summary uses a single surface; mobile summary collapses below the active step or appears as a stable compact strip |
| Progress | Use a text stepper: `1 Leistung`, `2 Stylist`, `3 Zeit`, `4 Kontakt`. Hide the stylist step entirely when skipped |
| Stability | Date rows, service rows, time chips, and CTA area keep stable dimensions across loading, hover, selected, disabled, and error states |

Do not use cards inside cards. Service rows, stylist options, date choices, and time slots are selectable rows/chips within the active step surface, not separate nested cards.

## Interaction Contract

| Step | Required behavior |
|------|-------------------|
| Service category | Show category chips first when more than one bookable category exists. Skip this control when only one category exists |
| Service selection | Show service name, duration, price label, and optional note. Selecting a service clears downstream stylist, date, slot, and submit state |
| Stylist selection | Skip entirely when one eligible stylist exists for the selected service. When multiple exist, show `Keine Praeferenz` first, then named stylists |
| Date selection | Show an initial compact set of 14 dates, within lead time and booking horizon. Allow moving forward within the configured horizon |
| Time selection | After a date is selected, show server-calculated time chips. Disabled or unavailable slots must not be selectable |
| Contact details | Require name, phone, and email. Note is optional. Keep entered data when moving backward or after stale-slot conflict |
| Submission | Disable submit while pending and keep button width stable. Server must re-check availability before insert |
| Conflict retry | Clear only the invalid slot, preserve service, stylist preference, selected date, name, phone, email, and note, then reload slots for the same date |

Keyboard and accessibility:

| Element | Contract |
|---------|----------|
| Chips and option rows | Native `button` or radio semantics; visible focus ring in accent color |
| Progress | Announce current step with text, not color alone |
| Errors | Place field errors directly under fields and provide a top-level summary for submit errors |
| Loading | Use text status such as `Freie Zeiten werden geladen...`; do not rely on spinner-only feedback |
| Reduced motion | Respect existing `useReducedMotion()` pattern; no required animation for task completion |

## State Contract

| State | Visual and copy contract |
|-------|--------------------------|
| Booking enabled, setup complete | Show guided booking flow immediately |
| Contact-only mode | Keep existing contact fallback; do not expose booking controls |
| Booking enabled, setup incomplete | Show public-friendly contact fallback with phone, WhatsApp, and email actions; no admin/setup wording |
| No service selected | Service step is active; CTA disabled or absent until service selection |
| One eligible stylist | Skip stylist step and record the resolved eligible staff path internally |
| Multiple eligible stylists | Show `Keine Praeferenz` and named stylist choices |
| No slots for selected date | Show empty state copy and keep date/stylist/service controls available |
| Slot lookup failed | Show `Freie Zeiten konnten nicht geladen werden. Bitte versuche es noch einmal.` with retry button `Zeiten neu laden` |
| Submit validation failed | Keep all fields visible with inline German errors |
| Stale slot conflict | Use the error state copy above; clear only selected slot and reload same-date slots |
| Success | Replace form with success panel and selected appointment summary |
